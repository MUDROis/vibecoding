Вики курса

Вопросы и ответы по каждому уроку. Ниже собраны самые частые вопросы учеников и развёрнутые ответы на них — по каждому уроку основного курса и бонусных модулей.

Урок 1 Что такое вайкодинг

У меня не открывается страница через Live Server. Что делать?
Проверьте, что файл называется index.html, находится в открытой папке, и вы нажали «Go Live» в правом нижнем углу VS Code. Если не помогает, перезапустите VS Code и попробуйте снова. Также убедитесь, что у вас установлена последняя версия расширения.
DeepSeek выдал код, но он не работает (ошибка в браузере).
Скопируйте ошибку и вставьте её обратно в DeepSeek с просьбой исправить. Или упростите промпт — попросите только базовую структуру. Можно также попросить AI объяснить, что означает ошибка.
Как загрузить файлы через VS Code, если я не вижу кнопку «Publish to GitHub»?
Убедитесь, что вы авторизовались в GitHub через VS Code (значок аккаунта в левом нижнем углу). Если кнопка всё равно не появляется, можно вручную создать репозиторий на GitHub, а затем в VS Code добавить удалённый репозиторий (через git remote add origin ...) и запушить.
Почему на сайте не отображается картинка?
Проверьте ссылку на плейсхолдер — она должна быть активна. Если нет, замените на другую (например, https://picsum.photos/200) или загрузите свою картинку в папку и укажите путь.
Что делать, если я забыл запушить изменения?
Откройте VS Code, сделайте коммит и нажмите «Sync Changes» или «Push». Сайт обновится через несколько минут.

Урок 2 Инструменты AI-разработчика

Чем Mimo отличается от Giga Code?
Mimo — терминальный агент, который может читать/писать файлы и управлять Git, подходит для длительных задач. Giga Code — расширение VS Code, которое анализирует проект и помогает понимать архитектуру, но сам код меняет реже.
Mimo не установился через npm. Что делать?
Убедитесь, что у вас установлен Node.js и npm. Проверьте версию: node -v и npm -v. Если npm не установлен, скачайте Node.js с официального сайта. Также попробуйте использовать официальный источник npm:
npm install -g @mimo-ai/cli --registry https://registry.npmjs.org
Мой трекер задач не сохраняет задачи после перезагрузки.
Убедитесь, что вы используете localStorage. Попросите AI добавить код для сохранения и загрузки задач из localStorage.
Какой AI лучше для написания кода?
DeepSeek и Qwen дают хорошие результаты для большинства задач. ChatGPT тоже подходит, но может быть медленнее. Лучше пробовать разные и выбирать тот, чей код вам понятнее.
Почему Mimo не появляется в боковой панели VS Code?
Mimo — терминальный агент, он не имеет панели в VS Code. Установите дополнительное расширение Mimo Code из магазина VS Code — оно добавит горячие клавиши для быстрого запуска Mimo в терминале.

Урок 3 Первый лендинг

Почему мои картинки не отображаются?
Проверьте пути к изображениям. Если используете локальные файлы, убедитесь, что они лежат в папке проекта. Если используете ссылки — проверьте, что они активны. Также можно использовать плейсхолдеры: https://via.placeholder.com/400x300 или https://picsum.photos/400/300.
Как добавить свой логотип вместо текста?
Попросите AI: «Замени текст логотипа на изображение logo.png, добавь код для логотипа в хедер». Или сделайте это вручную: вставьте тег <img> вместо текста.
Форма не работает, кнопка ничего не делает.
Убедитесь, что вы добавили JavaScript-код для обработки события submit и предотвратили перезагрузку страницы (event.preventDefault()). Если не получается — скопируйте ошибку в DeepSeek и попросите исправить.
Как добавить ссылки на соцсети в футер?
Используйте теги <a> с иконками Font Awesome:
<a href="#"><i class="fab fa-telegram"></i></a>
Лендинг выглядит плохо на телефоне. Как исправить?
Проверьте, что вы используете адаптивный дизайн (медиа-запросы). Попросите AI: «Добавь адаптивность для мобильных устройств, сделай карточки в одну колонку на телефоне».

Урок 4 Supabase

Почему данные не сохраняются в Supabase?
Проверьте:
  • Правильно ли вы скопировали URL и анонимный ключ (anon key).
  • Не блокируется ли отправка CORS (в консоли должна быть ошибка).
  • Правильно ли названа таблица и поля (регистр важен: requests).
  • Убедитесь, что в Supabase включены RLS (Row Level Security) — если включены, нужно создать политики.
Как посмотреть все заявки?
В Supabase откройте таблицу requests → вкладка Table Editor. Там вы увидите все строки.
Можно ли использовать Supabase бесплатно?
Да, бесплатный тариф включает 500 МБ базы данных, до 10 000 строк на таблицу и 100 000 запросов в день — этого достаточно для лендинга.
Как удалить заявку?
В Supabase наведите на строку в таблице → нажмите на три точки → Delete.
А если я хочу добавить ещё одно поле (например, комментарий)?
В Supabase откройте таблицу → нажмите Add column и добавьте новое поле. Затем обновите JavaScript-код, чтобы он отправлял это поле.

Урок 5 Git и резервное копирование

Я случайно удалил локальную папку с проектом. Как восстановить?
Если вы пушили изменения на GitHub, просто клонируйте репозиторий снова: git clone <URL>.
Что делать, если конфликт не разрешается автоматически?
Вручную выберите нужный код, удалите маркеры <<<<<<<, =======, >>>>>>>, сохраните и сделайте коммит.
Зачем нужны ветки, если можно работать в одной?
Ветки позволяют разрабатывать новые функции без риска сломать работающий код. Если что-то пойдёт не так, вы просто удалите ветку и начнёте заново.
Как сделать автоматический бэкап?
Мы настроили это через GitHub Actions — создайте файл .github/workflows/backup.yml с расписанием и командами commit/push. Подробная инструкция есть в уроке.
Что такое Pull Request и зачем он нужен?
Это запрос на слияние изменений из одной ветки в другую на GitHub. Он позволяет обсудить изменения перед слиянием, проверить код и убедиться, что всё работает.

Урок 6 GitHub Actions

Workflow упал с ошибкой «Permission denied». Что делать?
Убедитесь, что в настройках репозитория включён доступ GitHub Actions к GitHub Pages. Проверьте, что вы используете ${{ secrets.GITHUB_TOKEN }} — он автоматически создаётся GitHub и даёт права на деплой.
Проверка ссылок находит битые ссылки, но я не могу их исправить (внешний сайт недоступен).
Добавьте их в исключения: --skip "https://bad-site.com/*". Или настройте проверку так, чтобы она не падала, а только выводила предупреждение.
Уведомление в Telegram не приходит.
Проверьте, правильно ли заполнены секреты. Убедитесь, что вы написали боту /start. Также проверьте, что workflow действительно выполнился успешно (зелёная галочка).
Можно ли запускать workflow по расписанию?
Да, замените on: push на:
on:
  schedule:
    - cron: '0 0 * * *'
(каждый день в полночь). Но для деплоя обычно используют push.
Как добавить проверку нескольких страниц?
В команде linkinator можно указать несколько файлов или использовать --recurse для рекурсивной проверки.

Урок 7 Тестирование и отладка

Lighthouse показывает низкую оценку Performance. Что делать в первую очередь?
Чаще всего проблема в больших изображениях. Сожмите их (используйте TinyPNG или ImageOptim) или замените на WebP. Также проверьте, не подключены ли тяжёлые шрифты и скрипты.
Как узнать, какие изображения нужно оптимизировать?
В отчёте Lighthouse есть раздел «Opportunities» — там перечислены конкретные файлы и их размер. Также можно использовать вкладку Network в DevTools и посмотреть, какие ресурсы загружаются дольше всего.
A/B тест не работает, всегда показывает одну версию.
Проверьте код скрипта. Возможно, вы забыли удалить localStorage при тестировании — очистите кэш и localStorage в браузере (F12 → Application → Local Storage → Clear).
Нужно ли проводить A/B тестирование для учебного проекта?
Это дополнительная практика, но она даёт понимание, как работают настоящие маркетинговые эксперименты. Если у вас нет времени — пропустите этот шаг, но попробуйте хотя бы один раз.
Как проверить доступность (accessibility) сайта?
Используйте Lighthouse (вкладка Accessibility) или расширение axe DevTools. Исправляйте проблемы: добавляйте alt-тексты, убедитесь в контрастности цветов, используйте семантические теги.

Урок 8 SEO

Как долго индексируется сайт в Google?
Обычно от нескольких часов до нескольких дней. Вы можете ускорить процесс, запросив индексацию вручную в Search Console.
Что делать, если сайт не индексируется?
Проверьте robots.txt — не закрыт ли сайт от индексации. Убедитесь, что в мета-тегах нет <meta name="robots" content="noindex">. Проверьте, что sitemap.xml доступен по ссылке.
Как выбрать правильные ключевые слова?
Используйте сервисы: Яндекс Wordstat (wordstat.yandex.ru), Google Keyword Planner, или спросите у AI: «Какие ключевые слова лучше всего подходят для школы английского языка?».
Нужно ли писать статьи, если у меня только лендинг?
Статьи помогают привлекать трафик из поиска. Даже для небольшого проекта 3-5 статей значительно увеличат количество посетителей.
Могу ли я использовать AI для генерации всех статей?
Да, но обязательно редактируйте их: добавляйте примеры из своего опыта, проверяйте факты, делайте текст живым. Чистый AI-текст может быть распознан как некачественный.

Урок 9 Безопасность

У меня нет собственного домена, только GitHub Pages. Нужно ли настраивать SSL?
GitHub Pages автоматически даёт HTTPS для всех *.github.io доменов. Если у вас свой домен — нужно настроить через Cloudflare (бесплатно).
reCAPTCHA не работает, в консоли ошибка. Что делать?
Проверьте, что вы вставили правильный Site key. Убедитесь, что домен добавлен в настройках reCAPTCHA. Если используете локальный сервер, добавьте localhost в список доменов.
Как проверить, что SSL работает?
Откройте сайт по https://, в адресной строке должен быть замок. Кликните на замок → «Соединение безопасно».
Нужно ли соблюдать 152-ФЗ, если я не в России?
Если у вас есть пользователи из России — да. Если нет — достаточно общей политики конфиденциальности, которая информирует о сборе данных.
Что делать с галочкой согласия, если форма уже работает?
Просто добавьте чекбокс в форму и проверяйте его перед отправкой. Если галочка не стоит — показывайте сообщение «Пожалуйста, согласитесь на обработку данных».

Бонус 10 PWA

Зачем нужен PWA, если есть обычный сайт?
PWA даёт опыт нативного приложения: установка на телефон, работа офлайн, push-уведомления. Это повышает лояльность пользователей и упрощает доступ.
Моё PWA не устанавливается в Safari на iOS. Почему?
Для установки PWA на iOS нужно использовать Safari. Также нужно добавить специальные мета-теги для iOS:
<meta name="apple-mobile-web-app-capable" content="yes">
<link rel="apple-touch-icon" href="/images/icon-192.png">
Как обновить PWA после изменений?
Измените CACHE_NAME в sw.js (например, с my-site-v1 на my-site-v2). Новый Service Worker заменит старый и перекеширует файлы.
Почему service worker не регистрируется?
Проверьте:
  • Файл sw.js лежит в корне сайта (не в папке).
  • Пути к файлам в кеше указаны правильно.
  • Сайт работает по HTTPS (на локальном сервере тоже можно, но нужен localhost).
Можно ли использовать PWA без HTTPS?
Нет, для работы Service Worker требуется HTTPS (исключение — localhost для разработки).

Бонус 11 Telegram-бот

Уведомления не приходят. Что делать?
Проверьте:
  • Правильно ли вы скопировали токен и chat_id (без лишних пробелов).
  • Не блокируется ли запрос в Telegram (проверьте консоль браузера).
  • Убедитесь, что ваш бот активен (напишите ему любое сообщение).
Можно ли отправлять уведомления нескольким пользователям?
Да, добавьте несколько chat_id в массиве и отправляйте сообщение по очереди.
Как изменить текст сообщения?
Измените переменную message в коде. Можно добавить дату, ссылку на сайт, эмодзи и т.д.
Можно ли отправлять уведомления в групповой чат?
Да, добавьте бота в группу, получите chat_id группы (через @userinfobot или API) и используйте его.
Как сделать, чтобы бот отвечал на команды?
Это требует настройки webhook или polling. Для простых сценариев достаточно отправки уведомлений.

Бонус 12 Платежи

Можно ли принимать платежи без регистрации ИП?
Для ЮКассы и Stripe требуется регистрация бизнеса. Если вы хотите просто попробовать, используйте тестовый режим — никаких документов не нужно.
Чем отличается тестовый режим от боевого?
В тестовом режиме деньги не списываются, вы используете тестовые карты. В боевом режиме идут реальные списания, нужны реальные ключи и настройки.
Мой код не работает, в консоли ошибки. Что делать?
Проверьте:
  • Правильно ли вы скопировали ключи (без лишних пробелов).
  • Используете ли вы тестовые ключи в тестовом режиме.
  • Не блокируется ли запрос CORS (для локального тестирования используйте расширение браузера или настройте прокси).
Нужно ли мне настраивать webhook для простого сайта?
Не обязательно, но рекомендуется. Без webhook вы не узнаете о статусе платежа, если пользователь закроет страницу после оплаты. Это важно для автоматической активации доступа к курсу.
Как перейти из тестового режима в боевой?
В ЮКасса переключите переключатель «Тестовый режим» на «Боевой» в личном кабинете. В Stripe аналогично — выключите Test mode. После этого используйте боевые ключи.

Бонус 13 Docker

Docker не запускается на Windows. Что делать?
Убедитесь, что в BIOS включена виртуализация (VT-x/AMD-V). Включите WSL 2 (Windows Subsystem for Linux) и установите ядро Linux. Docker Desktop сам предложит настройки.
Как изменить порт, если 8080 занят?
Используйте другой порт в команде -p, например: -p 3000:80. Тогда сайт будет доступен по http://localhost:3000.
Можно ли использовать Docker без Docker Desktop?
Да, на Linux можно установить Docker Engine через пакетный менеджер (apt, yum и т.д.). На Windows и macOS Docker Desktop — самый простой способ.
Как обновить сайт, если он уже в контейнере?
Нужно пересобрать образ (docker build -t my-site .), остановить и удалить старый контейнер (docker rm -f my-site-container), затем запустить новый.
В чём разница между образом и контейнером?
Образ — это шаблон (как рецепт). Контейнер — это запущенный экземпляр (как готовое блюдо). Из одного образа можно создать множество контейнеров.

FAQ Общие вопросы по курсу

Нужно ли уметь программировать для курса?
Нет. Курс рассчитан на людей без технического образования. Всё, что нужно — уверенно пользоваться компьютером и интернетом.
Сколько времени занимает прохождение курса?
Основной курс — 3 недели (9 занятий по 2 часа). Бонусные модули можно пройти дополнительно, они не входят в основное расписание.
Есть ли поддержка после курса?
Да, у вас остаётся доступ к Telegram-чату, где можно задавать вопросы и общаться с другими учениками.
Что я получу в итоге?

Можно ли использовать эти знания для коммерческих проектов?
Да, конечно. Все инструменты и методы применимы для реальных бизнес-задач. Вы сможете создавать лендинги, сайты-визитки, интернет-магазины (с добавлением платежей) и другие продукты.

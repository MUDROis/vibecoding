Урок 2

Инструменты AI-разработчика:
выбираем и настраиваем

Устанавливаем AI-агентов, сравниваем нейросети и создаём трекер задач.

2 часа Начальный

Цели урока

После этого урока вы:

  • Познакомитесь с AI-агентами в VS Code — Mimo и Giga Code.
  • Установите и настроите оба агента (один через терминал, другой — через магазин расширений).
  • Научитесь сравнивать ответы разных нейросетей (DeepSeek, Qwen, ChatGPT).
  • Создадите свой первый интерактивный трекер задач (To‑Do List) с помощью AI.
  • Поймёте, как использовать AI для написания логики приложения.

Зачем нужны AI-агенты?

AI-агенты — это программы, которые интегрируются в VS Code и помогают вам писать код. Они могут:

  • Автодополнять код, когда вы печатаете.
  • Генерировать целые блоки кода по вашему описанию.
  • Анализировать ваш проект и давать рекомендации.
  • Исправлять ошибки и предлагать улучшения.

Проще говоря, это как «умный помощник», который сидит рядом и подсказывает, как писать код.

Сравнение AI-инструментов

DeepSeekОнлайн-чатПонимает русский, быстро генерирует кодБыстрые ответы и генерация с нуля

Ключевое различие: Mimo — «рукастый» помощник (сам вносит изменения в код). Giga Code — «аналитик» (помогает понять и структурировать проект).

Что такое трекер задач?

Трекер задач (To‑Do List) — простое веб-приложение, где можно:

  • Добавлять задачи с описанием.
  • Отмечать их как выполненные.
  • Удалять ненужные.

Это идеальный проект для изучения JavaScript (язык программирования, который делает сайты интерактивными) и работы с DOM (структура HTML-страницы, с которой код взаимодействует). Мы не будем писать код вручную — мы попросим AI сделать это за нас, а затем разберём полученный код.

Шаг 1: Установка Giga Code

Giga Code — расширение для VS Code, которое анализирует проект и помогает понимать архитектуру.

  1. В VS Code откройте Extensions (Ctrl+Shift+X).
  2. В поиске введите Giga AI или Giga Code и установите.
  3. После установки расширение появится на боковой панели.
  4. Зарегистрируйтесь через Gitverse.

Giga Code генерирует SPEC.md — описание архитектуры вашего проекта.

Шаг 2: Установка Node.js

Node.js — среда выполнения JavaScript вне браузера. Для установки Mimo нужен npm (менеджер пакетов), который идёт вместе с Node.js.

Аналогия: Node.js — «двигатель», а npm — «магазин приложений», откуда мы скачиваем Mimo.

  1. Скачайте nodejs.org → кнопка LTS.
  2. Запустите установщик, настройки по умолчанию. Убедитесь, что включена опция «Add to PATH».
  3. После установки перезапустите VS Code.

Проверка: откройте терминал и выполните:

node --version
v18.x.x или выше
npm --version
9.x.x или выше

Если вы видите номера версий — Node.js установлен успешно!

Шаг 3: Установка Mimo

Mimo — терминальный агент, который сам вносит изменения в код. Установите через npm.

Важно: откройте командную строку (cmd) или терминал (в VS Code: Terminal → New Terminal).

При первом запуске mimo init выберите MiMo Auto — бесплатный канал.

Шаг 4: Проверка работы агентов

Попросим Mimo создать файл README.md с планом разработки трекера задач.

В окне диалога вставьте промпт:

«Создай файл README.md для проекта 'Трекер задач'. В этом файле опиши план разработки: что должно быть в трекере, какие функции, как будет выглядеть интерфейс. Пиши простым языком, как для начинающих.»

Mimo создаст файл в папке проекта и заполнит его планом.

Откройте созданный файл и посмотрите, что получилось. Если план вам нравится — отлично! Если нет — попросите Mimo дополнить или изменить его.

Шаг 5: Сравнение нейросетей

Откройте DeepSeek, Qwen (опционально) и ChatGPT и задайте одинаковый промпт:

«Напиши HTML, CSS и JS для трекера задач. Пользователь может добавлять задачи, отмечать их как выполненные и удалять. Дизайн современный, тёмная тема, адаптивный. Весь код в одном файле.»

Или вставьте файл с планом, созданным в предыдущем шаге.

Сравните полученные ответы по:

  • Читаемости — насколько понятен код.
  • Работоспособности — работает ли сразу.
  • Качеству — какой код вы бы использовали.

Выберите лучший вариант для дальнейшей работы.

Шаг 6: Создание трекера задач

Генерация кода:

  • Возьмите лучший результат из предыдущего шага.
  • Если код не идеален, уточните промпт: «Добавь возможность редактирования задачи, храни задачи в localStorage.»
  • Скопируйте финальный код в VS Code, создайте index.html.

Запуск и тестирование:

  • Запустите Live Server и проверьте трекер.
  • Попробуйте добавить задачи, отметить выполнение, удалить.
  • Если что-то не работает, скопируйте ошибку и вставьте в DeepSeek для исправления.

Шаг 7: Доработка с помощью Mimo

Используйте Mimo, чтобы улучшить код. Например, в терминале выполните:

C:\> mimo

Впишите в окне диалога с агентом:

«Добавь в трекер задач стиль для выполненных задач (зачёркнутый текст), счётчик задач и функцию очистки всех задач.»

Mimo внесёт изменения прямо в файлы проекта.

Шаг 8: Публикация на GitHub Pages

Вы уже знаете, как это делать из первого урока.

  1. Создайте новый репозиторий (например, todo-tracker).
  2. Загрузите файлы через VS Code (Git init, add, commit, publish).
  3. Включите GitHub Pages в настройках репозитория (Settings → Pages → Branch: main, /root).
  4. Проверьте, что ваш трекер доступен по ссылке.

Отправьте ссылку на ваш трекер в чат группы.

Домашнее задание

  1. Добавьте в трекер новые функции: редактирование (двойной клик), фильтр (все/активные/выполненные). Используйте Mimo или DeepSeek.
  2. Закоммитьте и запушьте изменения.
  3. Пришлите ссылку на ваш трекер в чат группы.

Урок 5

Git — контроль версий и
резервное копирование

Не бойтесь экспериментов — Git поможет вернуться к работающей версии.

2 часа Средний

Цели урока

После этого урока вы:

  • Поймёте, зачем нужен Git и как он помогает не бояться экспериментов.
  • Научитесь сохранять (коммитить) изменения через VS Code.
  • Научитесь отправлять (push) и получать (pull) изменения из GitHub.
  • Узнаете, как откатить проект к более ранней версии.
  • Настроите автоматический бэкап кода с помощью GitHub Actions.

Что такое Git и зачем он нужен?

Git — это система, которая запоминает все изменения в ваших файлах. Представьте, что вы пишете книгу, и Git делает «снимки» каждый раз, когда вы сохраняетесь.

Аналогия: Git — это как «сохранить» в игре. Вы можете сохраниться перед сложным заданием, а если что-то пошло не так — загрузить предыдущее сохранение.

Зачем это нам?

  • Вы можете безопасно экспериментировать: если сломаете сайт, всегда можно вернуться к работающей версии.
  • Все ваши изменения хранятся в облаке (на GitHub), даже если компьютер сломается.
  • Вы можете работать над сайтом с нескольких компьютеров.

Основные понятия

Шаг 1: Инициализация Git в проекте

  1. Откройте ваш проект в VS Code.
  2. Убедитесь, что папка проекта открыта (File → Open Folder).
  3. В левой панели нажмите на значок Source Control (ветка) или нажмите Ctrl+Shift+G.
  4. Нажмите кнопку Initialize Repository — это создаст локальный репозиторий.

Если вы уже публиковали проект на GitHub на первом уроке, то Git уже инициализирован.

Шаг 2: Первый коммит (сохранение)

  1. Внесите какие-то изменения в файлы (например, исправьте текст на главной странице).
  2. В панели Source Control вы увидите изменённые файлы в разделе Changes.
  3. Наведите на файл и нажмите на значок + (Stage Changes) — это подготовит файл к коммиту. Можно нажать на + рядом с Changes, чтобы подготовить все файлы сразу.
  4. В поле сообщения (вверху) напишите короткое описание: например, «Исправил текст на главной».
  5. Нажмите на галочку Commit (или Ctrl+Enter).

Поздравляю! Вы сделали первый коммит. Теперь Git запомнил это состояние.

Шаг 3: Отправка (Push) на GitHub

Если репозиторий ещё не связан с GitHub:

  1. Нажмите на три точки (...) в панели Source Control → выберите Publish to GitHub.
  2. Выберите Public или Private и нажмите Publish.
  3. VS Code создаст репозиторий на GitHub и отправит ваш код.

Если репозиторий уже есть:

  1. Нажмите на кнопку Sync Changes (круглая стрелка) или в меню ... выберите Push.

После этого ваши изменения появятся на GitHub.

Шаг 4: Получение (Pull) изменений

Если вы работаете с нескольких компьютеров, вы можете загрузить последние изменения:

  1. Нажмите на кнопку Sync Changes (или в меню выберите Pull).
  2. VS Code загрузит новые коммиты из GitHub.

Совет: Всегда делайте Pull перед началом работы, чтобы получить последние изменения с GitHub.

Шаг 5: Использование AI для помощи с Git

Вы не обязаны запоминать все команды. Спросите у DeepSeek или Mimo, как выполнить ту или иную операцию.

Например:

  • «Как сделать коммит в VS Code через терминал?»
  • «Напиши инструкцию по откату к предыдущему коммиту через GitHub Desktop»
  • «Создай файл .gitignore для HTML-проекта, чтобы не загружать лишние папки»

Воспользуйтесь Mimo в терминале:

«Покажи мне все мои коммиты в этом проекте»

Шаг 6: Откат изменений

Если вы случайно сломали сайт, вы можете вернуть его к работающей версии.

Способ 1: Через GitHub Desktop (рекомендуется)

  1. Установите и откройте GitHub Desktop.
  2. Откройте ваш репозиторий.
  3. В верхней части нажмите на вкладку History (история) — вы увидите список всех коммитов.
  4. Найдите коммит, к которому хотите вернуться.
  5. Кликните правой кнопкой и выберите Revert changes in commit.
  6. Нажмите Fetch origin, чтобы отправить откат на GitHub.

Способ 2: Через веб-интерфейс GitHub

  1. Зайдите в репозиторий на GitHub.
  2. Нажмите на вкладку Commits (история коммитов).
  3. Найдите нужный коммит и нажмите Revert.

Откат через revert безопасен — он не удаляет историю, а создаёт новый коммит, который отменяет изменения.

Шаг 7: Автоматический бэкап через GitHub Actions

Настроим ежедневный бэкап вашего кода:

  1. В корне проекта создайте папку .github/workflows.
  2. Внутри создайте файл backup.yml и вставьте код (можно попросить AI сгенерировать):
name: Автоматический бэкап

on:
  schedule:
    - cron: '0 3 * * *'
  workflow_dispatch:

jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      - name: Commit and push changes
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add .
          git diff --quiet && git diff --staged --quiet || git commit -m "Автоматический бэкап"
          git push
  1. Закоммитьте и запушьте этот файл.
  2. Запустите бэкап вручную: GitHub → Actions → Run workflow.

Домашнее задание

  1. Сделайте коммит с любым изменением (например, добавьте новую страницу).
  2. Отправьте изменения на GitHub через Push.
  3. Попробуйте откатить одно изменение через GitHub Desktop или через веб-интерфейс GitHub (создайте revert-коммит).
  4. Настройте автоматический бэкап через GitHub Actions (если ещё не сделали).
  5. Напишите в чат краткий отчёт: что вы изменили, какой коммит откатили, работает ли бэкап.
